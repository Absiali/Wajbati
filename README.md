# 🏥 وجبتي - نظام طلب الوجبات للمستشفى

نظام متطور لإدارة طلبات الوجبات للمرضى النزلاء بالمستشفى.

## ✨ المميزات الرئيسية

### 👨‍⚕️ لوحة الإدارة (Admin Panel)
- ✅ إدارة خيارات الفطور والغداء والعشاء
- ✅ إضافة وتعديل وحذف الوجبات
- ✅ عداد فوري لعدد الوجبات في كل فئة
- ✅ نظام إشعارات سريع
- ✅ تصدير البيانات بصيغة JSON
- ✅ عرض الإحصائيات الكاملة

### 🏃 واجهة المريض (Patient Interface)
- 🍽️ طلب الوجبات بسهولة
- 📋 معاينة الخيارات المتاحة
- 🔔 تأكيد الطلب والإشعارات
- 📱 تصميم متوافق مع الهاتف

### 💾 نظام التخزين
- 🔐 حفظ البيانات محلياً (LocalStorage)
- 📊 إمكانية التصدير والاستيراد
- 🔄 مزامنة فورية بين الصفحات

## 🚀 كيفية الاستخدام

### للإداريين:
1. افتح `admin.html` في المتصفح
2. أضف خيارات الوجبات لكل فئة (فطور، غداء، عشاء)
3. استخدم الأزرار للتعديل أو الحذف
4. صدّر البيانات للنسخ الاحتياطي

### للمرضى:
1. افتح `patient.html` أو الصفحة الرئيسية
2. اختر الوجبة المرغوبة
3. أكد الطلب
4. ستتلقى تأكيداً للطلب

## 💡 الميزات المتقدمة

### نظام الإشعارات
- ✅ إشعارات النجاح (أخضر)
- ⚠️ إشعارات التحذير (برتقالي)
- ❌ إشعارات الخطأ (أحمر)
- ℹ️ إشعارات معلومات (أزرق)

- 🔐 حماية من XSS (Cross-Site Scripting)
- ✔️ التحقق من صحة الإدخال
- 🚫 منع تكرار الوجبات

### التوافقية
- 📱 تصميم متجاوب (Responsive)
- 🖥️ يعمل على جميع الأجهزة
- ⌨️ دعم لوحة المفاتيح

## 🔧 التثبيت والتشغيل

### الطريقة الأولى: النسخ المباشر
```bash
# انسخ الملفات إلى مجلد
cd path/to/wajbati
# افتح admin.html أو index.html في المتصفح
```

### الطريقة الثانية: استخدام خادم محلي
```bash
# استخدم Python 3
python -m http.server 8000

# أو Node.js
npx http-server

# ثم افتح http://localhost:8000
```

## 🐛 استكشاف الأخطاء

### المشكلة: البيانات لا تُحفظ
**الحل:** تأكد من أن المتصفح يدعم LocalStorage

**الحل:** استخدم خادم محلي بدلاً من فتح الملف مباشرة

**الحل:** تأكد من تفعيل JavaScript في المتصفح
